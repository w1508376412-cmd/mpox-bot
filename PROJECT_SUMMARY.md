# 猴痘知识问答机器人 - 项目总结

## 项目概述

这是一个基于RAG（检索增强生成）技术的猴痘/mpox健康科普问答系统，使用权威来源的知识库而非直接训练模型，确保信息的准确性和可追溯性。

## 核心特性

### 1. RAG架构
- **不训练模型**：使用向量检索 + 大模型生成，避免模型幻觉
- **权威来源**：WHO、中国疾控中心、CDC等官方资料
- **可追溯性**：每个回答都标注具体来源和发布日期

### 2. 安全分诊系统
- **风险分类**：自动识别一般科普、症状咨询、高风险情况
- **医疗免责**：明确说明不能替代医生诊断
- **就医建议**：高风险症状自动建议就医

### 3. 地区适配
- **多地区支持**：中国、美国、全球
- **政策差异**：根据地区提供相应的就医指导
- **优先级排序**：优先返回用户所在地区的权威信息

## 技术架构

```
用户问题
    ↓
风险分类（safety.py）
    ↓
向量检索（retriever.py + pgvector）
    ↓
上下文格式化
    ↓
大模型生成（generator.py + OpenAI API）
    ↓
安全提示 + 回答 + 来源
```

## 核心模块说明

### 后端模块

| 模块 | 功能 | 关键技术 |
|------|------|----------|
| `crawler.py` | 数据采集 | requests, trafilatura |
| `chunker.py` | 文档切片 | 600字/片段，80字重叠 |
| `embedder.py` | 向量生成 | OpenAI text-embedding-3-small |
| `retriever.py` | 向量检索 | pgvector 余弦相似度 |
| `generator.py` | 回答生成 | OpenAI GPT-4o-mini |
| `safety.py` | 安全分诊 | 关键词匹配 + 规则引擎 |
| `main.py` | API服务 | FastAPI |

### 数据库设计

**documents表**：存储原始文档元数据
- id, title, source, url, publish_date, region

**chunks表**：存储知识片段和向量
- id, document_id, content, topic, embedding(1536维)
- source, url, publish_date, region
- is_active, version（用于版本管理）

### 前端设计

- 单页面应用，原生HTML/CSS/JavaScript
- 实时问答交互
- 风险等级标识（高风险/症状/政策/科普）
- 来源展示和追溯
- 医疗免责声明

## 安全规则

### 1. 风险分类规则

| 风险类型 | 触发条件 | 处理方式 |
|---------|---------|---------|
| high_risk | 呼吸困难、意识不清、孕妇、免疫低下 | 明确建议尽快就医 |
| symptom_risk | 皮疹、发烧、淋巴结肿大 | 说明无法诊断，建议就医 |
| policy | 就医、隔离、检测 | 优先引用本地疾控 |
| general | 科普问题 | 正常回答 |

### 2. 回答规则

- ✅ 只能根据检索到的资料回答
- ✅ 不能说"你确诊了"或"你没有感染"
- ✅ 症状相关问题必须建议就医
- ✅ 每个结论必须标注来源
- ✅ 资料不足时直接说明

### 3. 医疗免责

每个回答都包含免责声明：
> 本机器人仅提供猴痘/mpox健康科普信息，不能替代医生诊断或治疗。如出现新发或原因不明皮疹、发热、淋巴结肿大，或有可疑接触史，请咨询医疗机构或当地疾控部门。

## 成本估算

### OpenAI API费用

| 操作 | 模型 | 单次成本 | 说明 |
|------|------|---------|------|
| 向量生成 | text-embedding-3-small | ~$0.0001/片段 | 一次性处理 |
| 问答 | gpt-4o-mini | ~$0.001/次 | 每次用户提问 |

**初始化成本**：约$0.05（500个片段）
**运行成本**：约$1/1000次问答

### 数据库

- PostgreSQL + pgvector：免费开源
- 存储需求：约100MB（500个片段）

## 维护和更新

### 定期任务

1. **每周更新**：检查权威来源是否有新内容
2. **每月评估**：运行测试集，检查回答质量
3. **版本管理**：保留历史版本，支持回滚

### 更新流程

```bash
# 1. 抓取最新数据
cd backend
python crawler.py

# 2. 对比变化
# 如果有变化，重新处理
python process_data.py

# 3. 运行测试
cd ../eval
python run_eval.py

# 4. 如果测试通过，标记为生产版本
# 如果测试失败，回滚到上一版本
```

## 注意事项

### 法律合规

- ✅ 明确医疗免责声明
- ✅ 不做诊断，只提供科普
- ✅ 建议用户咨询专业医生
- ✅ 使用权威公开资料

### 数据隐私

- ✅ 不收集用户个人信息
- ✅ 不存储用户提问内容
- ✅ API调用日志定期清理

### 技术风险

- ⚠️ OpenAI API可用性依赖
- ⚠️ 向量检索可能遗漏相关内容
- ⚠️ 大模型可能产生不准确回答
- ⚠️ 权威来源更新延迟

### 缓解措施

1. **多来源验证**：同时检索多个权威来源
2. **置信度阈值**：相似度过低时拒绝回答
3. **人工审核**：定期抽查回答质量
4. **版本控制**：保留历史版本，支持快速回滚

## 总结

这个猴痘知识问答机器人采用RAG架构，通过向量检索和大模型生成，提供基于权威来源的健康科普服务。核心优势是：

1. **准确性**：基于权威来源，可追溯
2. **安全性**：风险分诊，医疗免责
3. **可维护性**：易于更新，版本管理
4. **低成本**：API调用费用低，无需训练模型

适合作为健康科普工具，但不能替代专业医疗诊断。
